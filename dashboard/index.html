Run both apps locally for rapid iteration — no Docker needed at this stage.

# Terminal 1 — Anelia cd apps/anelia && cp .env.example .env npm install && npm run dev # Terminal 2 — School cd apps/school && cp .env.example .env npm install && npm run dev # Test curl http://localhost:3001/health curl http://localhost:3002/health
🔧 pm2 as a Docker alternative

If Docker feels heavy on a 2 GB box, pm2 manages both Node processes directly with auto-restart on crash.

npm install -g pm2 pm2 start apps/anelia/index.js --name anelia pm2 start apps/school/index.js --name school pm2 save && pm2 startup # survive reboots pm2 logs anelia # tail logs pm2 restart school # rolling restart

🤖 Quick Claude Code prompts — copy & paste
Bootstrap & health-check
Verify docker compose is running, cloudflared is active, and both /health endpoints return 200. Fix anything that is broken. Repo is at /root/hq.
Add an Airtable route (Anelia)
In /root/hq/apps/anelia/index.js add POST /api/items: - Accept JSON { name, description } - Write a new record to Airtable base $AIRTABLE_BASE_ID - Return the created record with status 201 Use fetch (Node 18+), read creds from process.env. Rebuild the container after.
Deploy after a code change
I just changed /root/hq/apps/school/index.js. Rebuild: docker compose up -d --build school Confirm GET /health returns 200. Then run: ./scripts/deploy.sh school
Check Airtable headroom
Count total records in the school Airtable base (base ID: $AIRTABLE_BASE_ID, key: $AIRTABLE_API_KEY). Warn if count exceeds 800. Free tier cap is 1,000.
